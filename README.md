<table>

<td width="50%">

### Static Analysis

Examining malware without execution to understand structure and intent:

- **PE/ELF Analysis** â€” Header parsing, section analysis, import/export tables
- **Disassembly** â€” x86/x64 instruction decoding with Ghidra & IDA Pro
- **Code Pattern Recognition** â€” Identifying malicious patterns, packers, crypters
- **String Analysis** â€” Extracting IOCs, URLs, C2 addresses
- **YARA Rules** â€” Writing detection signatures for threat hunting
- **AI-Assisted Analysis** â€” Using LLMs for automated code understanding

</td>
<td width="50%">

### Dynamic Analysis

Observing malware behavior during controlled execution:

- **Sandboxing** â€” Isolated execution environments for safe analysis
- **API Monitoring** â€” Hooking system calls, tracking file/registry operations
- **Network Analysis** â€” Capturing C2 traffic, protocol analysis
- **Process Inspection** â€” Memory dumps, injection detection
- **Behavioral Profiling** â€” Building execution timelines and graphs

</td>
</tr>
<tr>
<td width="50%">

### Memory Forensics

Analyzing volatile memory for evidence and artifacts:

- **Volatility Framework** â€” Memory image analysis and artifact extraction
- **Process Archaeology** â€” Hidden process detection, DKOM analysis
- **Injection Detection** â€” Identifying code injection, hollowing, hooking
- **Rootkit Analysis** â€” Kernel-mode malware investigation
- **Credential Extraction** â€” Memory-resident secrets and tokens

</td>

### Reverse Engineering

Deep binary analysis and code understanding:

- **Assembly Analysis** â€” x86/x64, ARM instruction sets
- **Control Flow Recovery** â€” Reconstructing program logic
- **Deobfuscation** â€” Unpacking, decrypting, deobfuscating protected code
- **Protocol Reverse Engineering** â€” Understanding proprietary protocols
- **Cryptographic Analysis** â€” Identifying and analyzing crypto implementations
